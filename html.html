I### Mozilla Firefox: 1. **Using Keyboard Shortcuts:** - Press Ctrl + Shift + I on Windows or Linux. - Press Cmd + Option + I on macOS. 2. **Using the Menu:** - Click on the three horizontal lines in the top-right corner to open the menu. - Go to "Web Developer" and then select "Web Console." 3. **Right-Click Method:** - Right-click on any element on the webpage. - Select "Inspect Element" from the context menu. - This will open the developer console at the bottom of the browser window.

### Safari: 1. **Using Keyboard Shortcuts:** - Press Option + Command + I to open the developer console. 2. **Using the Menu:** - Open Safari and go to the menu bar. - Click on "Develop" in the menu bar. - Select "Show Web Inspector." 3. **Right-Click Method:** - Right-click on any element on the webpage. - Select "Inspect Element" from the context menu. - This will open the developer console at the bottom of the browser window.

What way of including internal links into the HTML page you would consider generally preferable: absolute or relative links? Describe use cases for choosing absolute links and choosing relative links.

Use Cases for Choosing Absolute Links: 1. When linking to external websites or resources located on different domains. 2. When linking to resources that are hosted on a different server or subdomain within the same website. 3. When you need to ensure that the link always points to a specific location regardless of the current page's URL.

Use Cases for Choosing Relative Links: 1. When linking to pages or resources within the same website or directory structure. 2. When creating links between different pages or sections of a website that are likely to be moved or reorganized. 3. When you want to simplify the maintenance of links by avoiding the need to update URLs if the website structure changes.

Ultimately, the choice between absolute and relative links depends on the specific requirements of your website and how likely the URLs are to change in the future. It's generally recommended to use relative links for internal navigation within a website for better flexibility and maintainability.

Research and write about image formats used in web-development. What is the difference between JPG, GIF and PNG formats? What is WEBP image type? What is SVG?

Image formats commonly used in web development include JPG, GIF, PNG, WEBP, and SVG. Each format has unique characteristics and is suited for different types of images.

JJPG is a widely used image format for photographs and complex images. It uses lossy compression to reduce file size while maintaining reasonable image quality. JPG is best suited for images with many colors or gradients, such as photographs.

GIF is popular for simple images, animations, and graphics with fewer colors. It supports transparency and animation, making it ideal for logos, icons, and simple animations due to its small file size and support for transparency.

PNG is commonly used for logos, illustrations, and images requiring a transparent background.

WEBP supports transparency and animation, making it an excellent choice for optimizing website performance.

SVG is a vector-based image format that uses XML to define graphics. SVG images are resolution-independent and can be scaled without losing quality. They are perfect for logos, icons, and illustrations that need to be displayed at various sizes on the web.
